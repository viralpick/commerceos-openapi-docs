Email Node

설명

Email 노드는 전체적인 워크플로우의 실행 중에 SMTP를 통해 이메일을 발송합니다. 제목과 본문에서 {{variable}} 문법을 지원해 동적 콘텐츠를 포함할 수 있습니다.

사용 시점

워크플로우에서 결과, 알림, 경보를 이메일로 전달해야 할 때 사용합니다. 일별 리포트 발송, 업스트림 Agent 노드에서 감지된 이상값 알림, 수집된 데이터의 요약 전달 등에 활용할 수 있습니다.

설정

Node information + Recipient Configuration

필드필수설명
NameYes워크플로우 내에서 노드를 고유하게 식별하는 이름입니다.
DescriptionNo노드의 역할을 간략하게 설명하는 텍스트입니다.
Recipient(s)Yes수신자 이메일 주소를 쉼표로 구분해 입력합니다. {{variable}}을 사용해 동적 값을 지정할 수 있습니다.
CCNoToggle
활성화시 참조 이메일을 추가할 수 있습니다.
BCCNoToggle
활성화시 숨은 참조 이메일을 추가할 수 있습니다.

Email Content Configuration

필드필수설명
SubjectYes이메일 제목입니다. {{variable}}을 사용해 동적 값을 지정할 수 있습니다.
MessageYes이메일 본문입니다. Plain Text와 HTML 콘텐츠 유형 간 전환이 가능하며, {{variable}}을 사용해 동적 값을 지정할 수 있습니다.

사용 팁

  • 업스트림 Agent 노드의 결과를 발송할 때, 원시 출력을 그대로 사용하지 말고 Variable 기능을 사용해 특정 출력 필드만 본문 템플릿에 포함하세요.
  • Content Type이 Plain Text인 경우 HTML 태그가 일반 텍스트로 렌더링됩니다 — 서식이 필요한 경우 HTML로 전환하세요.

✅ Examples:

분석 결과 알림 (Plain Text):

Subject: {{brand}} 분석 완료

Body:

{{brand}} 에 대한 분석이 완료되었습니다.

브랜드 분석 리포트 (HTML):

Subject: {{brand}} 분석 리포트

Body:

<h2>{{brand}} 분석 리포트</h2>
<p>{{summary}}</p>

자주 발생하는 오류

  • recipient(s) 필드에 유효하지 않거나 도달할 수 없는 이메일 주소가 있으면 노드가 실패합니다. 배포 전에 수신자 주소를 확인하세요.
  • 잘못된 {{variable}} 참조는 처리되지 않습니다. Variable 드롭다운에서 지원하는 정확한 필드명을 사용하세요.
