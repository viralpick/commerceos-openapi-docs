# Data Preview

## 개요

Data Preview는 파이프라인 상에서 특정 노드를 기준으로 일부 실행을 수행하여,  
각 단계의 데이터를 **샘플 단위로 미리 확인할 수 있는 기능**이다.

이 기능은 전체 파이프라인 실행 전에 데이터 흐름과 구조를 검증하기 위한 목적으로 제공된다.

- 최대 **100행(Row)**까지 샘플 데이터를 확인할 수 있다
- 저장(Sink) 노드는 실제 쓰기를 수행하지 않고 **스킵(Skip)** 처리된다

---

## 주요 개념

- **프리뷰 트리거 노드**  
  사용자가 선택하여 Data Preview를 실행한 기준 노드

- **활성 노드**  
  현재 Data Preview 패널에서 선택되어 상세 결과가 표시되는 노드

- **프리뷰 결과 맵 (`previewMap`)**  
  노드별 실행 결과(상태, 데이터, 오류 등)를 포함하는 응답 구조

- **실행 순서 (`executionOrder`)**  
  서버가 프리뷰 실행 시 노드를 처리한 순서

---

## 사용 방법

### 1. 노드 선택

1. 파이프라인 편집 화면으로 이동
2. 캔버스에서 기준 노드 선택
3. 우측 편집 패널 확인

---

### 2. Data Preview 실행

- 우측 패널 하단의 **Data Preview 버튼 클릭**
- 실행 시 하단 패널에 결과 표시

---

## 버튼 활성 조건

다음 조건을 만족해야 Data Preview를 실행할 수 있다.

- 업스트림에 다음 중 하나가 존재해야 함
  - `Data Retrieval (DB)`
  - `Data Retrieval (HTTP)`

다음 조건에 해당하면 버튼이 비활성화된다.

- 노드 타입이 `custom` 또는 `custom-node`
- 업스트림에 `custom` 노드가 존재
- 데이터 소스 노드가 연결되지 않음

---

## 결과 패널 구성

### 헤더

- 성공 / 실패 / 스킵 개수 요약
- 활성 노드 정보
- 패널 접기/펼치기

---

### 본문

- 좌측: 노드 목록
- 우측: 선택된 노드의 상태 및 데이터

---

## 상태 정의

| 상태        | 설명                             |
| ----------- | -------------------------------- |
| **success** | 정상적으로 데이터 반환           |
| **fail**    | 실행 중 오류 발생                |
| **stale**   | 이전 결과이며 현재 설정과 불일치 |
| **sink**    | 저장 노드로 인해 실행이 스킵됨   |

---

## 데이터 확인

- 최대 100행까지 표시
- 페이지 크기: 5 / 10 / 20 / 50
- 페이지네이션 지원

---

## 자주 발생하는 상황

- 버튼이 비활성 → 데이터 소스 연결 확인 필요
- stale 상태 → 설정 변경 후 재실행 필요
- sink 상태 → 저장 노드 특성상 정상 동작
