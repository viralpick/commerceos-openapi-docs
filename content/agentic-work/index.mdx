# Agentic Work

## Agentic Work란?

Agentic Work는 AgentOS의 핵심 실행 엔진입니다. AI 기반 프로세스를 구축하고, 배포하고, 커머스 운영 전반에 걸쳐 자동으로 실행할 수 있는 도구를 제공합니다.

AI Chat이 즉흥적인 질문과 탐색에 적합한 도구라면, Agentic Work는 구조화되고 반복 가능한 작업들의 실행을 위한 공간입니다. 비즈니스 로직을 한 번 정의해두면 이후에는 일관되게 적용됩니다 — 스케줄에 따라 가격 데이터를 수집하거나, 조건에 따라 알림을 라우팅하거나, 결과를 변환해 데이터 레이어에 저장하는 작업들이 여기서 이루어집니다.

Agentic Work는 두 가지 핵심 개념을 중심으로 구성됩니다. **Workflows**와 **Automations**입니다. Workflow는 무엇을 해야 하는지를 정의하고, Automation은 언제 해야 하는지를 정의합니다.

> **주요 구분:** Agent Builder는 *AI 에이전트와 비즈니스 로직*을 오케스트레이션합니다. 원시 데이터 ETL(데이터베이스 수집, 데이터 변환)이 필요한 경우에는 Pipeline의 별도 **Pipeline Builder**를 사용하세요.

### Workflows (워크플로우)

워크플로우는 시각적 캔버스를 통해 정의되는 독립적인 프로세스입니다. 각 워크플로우는 AI 추론, 코드 실행, 조건 분기, 사용자 알림, 데이터 작업 등의 연결된 단계들로 구성되며, Start 노드에서 End 노드까지 방향 그래프 형태로 배치됩니다.

예를 들어, 하나의 워크플로우가 상품명을 입력으로 받아 여러 쇼핑 플랫폼에서 현재 가격을 조회하고, 내부 데이터와 비교한 뒤 구조화된 요약을 출력할 수 있습니다. 이 시퀀스의 각 단계가 워크플로우의 노드로 표현되고, 노드 간의 연결이 실행 순서를 결정합니다.

워크플로우는 버전 관리가 포함된 draft-and-deploy 생애주기를 따릅니다. 이를 통해 팀은 프로덕션에서 안정적인 버전을 유지하면서 워크플로우 로직을 반복적으로 개선할 수 있습니다. 배포된 워크플로우는 필요 시 수동으로 실행하거나, 자동화에 연결하여 운영할 수 있습니다.

Workflow를 만들고, 설정하고, 관리하는 방법에 대한 자세한 내용은 [Workflows](/agentic-work/workflows)를 참고하세요.

### Automations (자동화)

Automation은 조건에 반응하여 작업을 실행하는 규칙입니다. **If / Then** 구조를 따릅니다. 지정된 트리거(시간 기반 스케줄 또는 시스템 이벤트)가 충족되면 AgentOS가 연결된 작업을 실행합니다.

Automation은 Workflow 실행만으로 제한되지 않습니다. 리포트 생성, Excel 출력, 대시보드 업데이트, 데이터 수집 작업도 시작할 수 있어 플랫폼 전반의 중앙 스케줄링 및 오케스트레이션 메커니즘 역할을 합니다.

트리거 설정, 범위 지정, Automation 모니터링에 대한 자세한 내용은 [Automations](/agentic-work/automations)를 참조하세요.

## AgentOS에서 Agentic Work의 역할

Agentic Work는 플랫폼의 다른 구성 요소를 연결하는 실행 레이어입니다. Agent는 AgentOS의 구조화된 비즈니스 객체 데이터 레이어인 [**Ontology**](/ontology)에서 읽고 쓸 수 있습니다. Agent 출력은 실시간 모니터링 및 분석을 위한 **Dashboard**로, 또는 대규모 데이터 처리 및 변환을 위한 **Pipeline**으로 전달될 수 있습니다. 배포된 Agent는 **Agentic AI** 채팅 인터페이스에서도 호출할 수 있어, 대화형 쿼리가 배후에서 전체 워크플로우 실행을 트리거할 수 있습니다.

AI Chat이 대화적이고 탐색적인 반면, Agentic Work는 구조적이고 운영 중심적입니다. 두 기능은 함께 커머스 운영 전반에 걸쳐 임시 조회의 유연성과 자동화된 반복 프로세스의 안정성을 모두 제공합니다.
