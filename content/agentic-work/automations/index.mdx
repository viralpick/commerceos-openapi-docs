# Automations

## 개요

Automation은 트리거와 작업을 연결하여 수동 개입 없이 작업이 실행되도록 합니다. 각 Automation은 **If / Then** 구조를 따릅니다. Automation을 활성화하는 조건을 정의하고, 해당 조건이 충족될 때 실행되어야 할 작업을 지정합니다. Automation은 Agentic Work 섹션의 Automations 탭에서 관리됩니다.

---

## Automations 목록

Automations 탭을 열면 두 섹션이 표시됩니다: **My Automations**와 **Team Automations**.

**My Automations**에는 사용자가 생성했거나 편집 권한이 있는 모든 Automation이 표시됩니다. 목록의 각 항목은 Automation 이름, 진행 상태 표시기, 마지막 실행 날짜, 범위를 보여줍니다. 각 항목 왼쪽의 토글 스위치는 Automation의 활성화 여부를 제어합니다. 토글이 켜져 있으면 Automation이 활성 상태이며 트리거 조건이 충족될 때 실행됩니다. 토글이 꺼져 있으면 Automation이 일시 중지되어 실행되지 않습니다. 일부 항목에는 상태 배지가 표시될 수 있습니다 — 예를 들어 "Failed" 배지는 가장 최근 실행에서 오류가 발생했음을 나타냅니다.

**Team Automations**에는 그룹의 다른 멤버가 생성한 Automation이 표시됩니다. 보기 및 복제 권한이 있지만 직접 편집할 수는 없습니다. 테이블에는 Automation 이름, 진행 상태 표시기, 마지막 실행 날짜, 생성자가 표시됩니다.

각 섹션 위에는 이름으로 Automation을 필터링하는 검색창이 있습니다.

### Progress 표시기

각 Automation은 Progress 열에 Automation의 설정 상태를 나타내는 아이콘 집합을 표시합니다. 이 아이콘들은 트리거가 설정되었는지, 조건이 연결되었는지, 작업이 할당되었는지를 나타냅니다. 이 표시기 중 하나라도 없는 Automation은 완전히 설정되지 않은 것이며 실행되지 않습니다.

### 캘린더 사이드바

Automations 목록 오른쪽에 날짜 기반 Automation 활동 보기를 제공하는 캘린더 사이드바가 있습니다. 날짜를 선택하면 세 가지 카테고리로 구성된 Automation을 볼 수 있습니다: Running(현재 실행 중), Pending(예약 대기 중), Completed(선택한 날짜에 완료됨). 이 사이드바는 개별 항목을 열지 않고도 일별 Automation 활동을 빠르게 파악할 수 있게 해줍니다.

## Automation 생성

새 Automation을 생성하려면 Automations 목록 우측 상단의 **+ New Automation** 버튼을 클릭합니다. 빈 If/Then 캔버스가 있는 Automation 에디터가 열립니다. 페이지 상단의 기본 이름 옆에 있는 편집 아이콘을 클릭하여 Automation 이름을 변경할 수 있습니다.

에디터는 세 가지 주요 영역으로 구성됩니다: 트리거를 정의하는 상단의 **If** 섹션, 실행할 작업을 할당하는 하단의 **Then** 섹션, 헤더의 **Scope** 드롭다운.

## 트리거 정의 (If)

If 섹션의 **Add Condition** 버튼을 클릭하여 트리거 유형을 선택합니다. 두 가지 트리거 유형이 있습니다: **Event**와 **Schedule**.

### Event 트리거

Event 트리거는 Ontology 데이터 내에서 무언가가 발생했을 때 Automation을 실행합니다. Event를 선택하면 두 가지 필드가 있는 설정 모달이 열립니다: **Event Type**과 **Object Selection**.

Event Type은 Automation을 활성화할 변경의 종류를 결정합니다. 현재 사용 가능한 이벤트 유형은 "New row created"로, 선택된 Ontology 객체에 새 행이 삽입될 때마다 실행됩니다.

Object Selection은 모니터링할 Ontology 객체를 선택합니다. 선택된 객체에서 지정된 이벤트가 발생하면 Automation이 트리거되어 할당된 작업을 실행합니다.

저장하면 If 블록에 이벤트 설정이 요약으로 표시됩니다 — 예를 들어 "New row created"와 객체 이름이 함께 표시됩니다.

### Schedule 트리거

Schedule 트리거는 정의된 시간 간격에 따라 Automation을 실행합니다. Schedule을 선택하면 다음 옵션이 있는 설정 모달이 열립니다.

**Execution Frequency**는 Automation이 실행되는 빈도를 결정합니다. 세 가지 빈도 옵션이 있습니다: Hourly, Daily, Weekly.

**Hourly**로 설정하면 간격 값(예: 매 1시간 또는 매 3시간)을 지정합니다. 선택적인 **Time Specification** 토글을 사용하여 각 시간 내에서 Automation이 실행되어야 할 정확한 분을 정의할 수 있습니다.

**Daily**로 설정하면 간격 값(예: 매 1일 또는 매 2일)을 지정합니다. Time Specification 토글을 활성화하면 시간 필드와 타임존 선택기가 나타나 Automation이 실행되어야 할 정확한 시간(예: Asia/Seoul 기준 오전 08:30:00)을 설정할 수 있습니다.

**Weekly**로 설정하면 각 요일(일요일~토요일)에 대한 체크박스 토글로 하루 이상을 선택합니다. 시간 필드와 타임존 선택기를 통해 선택된 각 날짜의 정확한 실행 시간을 정의할 수 있습니다.

저장하면 If 블록에 스케줄 요약이 표시됩니다 — 예를 들어 "Every 1 hour" 또는 "Every day at 08:30 AM".

### 추가 조건 설정

If와 Then 섹션 사이의 **+** 버튼을 사용하여 Automation에 추가 조건을 설정할 수 있습니다. 이를 통해 여러 트리거를 조합하거나 Automation이 실행되어야 하는 시점을 더욱 세밀하게 정의할 수 있습니다.

## 작업 지정 (Then)

Then 섹션의 **Add Work** 버튼을 클릭해 Automation이 실행할 Workflow를 선택합니다. 선택 패널에는 접근 가능한 모든 Workflow가 **System Workflows**, **My Workflows**, **Team Workflows** 세 가지 카테고리로 표시됩니다 — Workflows 목록 페이지와 동일한 구성입니다.

### Work 액션 설정

Workflow를 선택하면 다음 필드가 있는 설정 모달이 열립니다.

**Work**는 배포된 Workflow 목록이 표시되는 필수 드롭다운입니다. 각 항목에는 Workflow 이름과 엔드포인트 경로가 함께 표시되어, 여러 Workflow 중에서도 원하는 것을 쉽게 찾을 수 있습니다. Automation이 동작하려면 반드시 여기서 Workflow를 선택해야 합니다.

**Additional Input**은 선택 사항인 체크박스로, 활성화하면 보충 지시나 컨텍스트를 입력할 수 있는 텍스트 영역이 나타납니다. 여기에 입력한 텍스트는 실행 시점에 Workflow로 전달됩니다. 동일한 Workflow가 트리거에 따라 다른 시나리오를 처리해야 할 때 유용합니다.

**Receive Execution Results via Email**은 선택적 체크박스로, 활성화하면 Automation 생성자에게 실행 결과가 포함된 이메일 알림이 발송됩니다. 무인으로 실행되는 Automation에 특히 유용하며, 플랫폼을 직접 확인하지 않고도 결과를 파악할 수 있습니다.

**Save**를 클릭해 설정을 저장합니다. 저장되면 Then 블록에 선택한 Workflow 이름이 캔버스에 표시됩니다.

## Scope

모든 Automation에는 에디터 우측 상단의 **Scope** 드롭다운으로 제어되는 범위 설정이 있습니다. 두 가지 옵션이 있습니다.

**Only Me**는 Automation을 본인만 볼 수 있도록 제한합니다. My Automations 목록에만 표시됩니다.

**Entire Group**은 그룹의 모든 멤버가 Automation을 볼 수 있게 합니다. 이 범위로 설정하면 다른 그룹 멤버의 Team Automations 섹션에 표시되어 보기 및 복제 권한이 부여됩니다.

## Automation 편집

If 블록과 Then 블록 모두 설정 요약 옆에 편집(연필) 및 삭제(휴지통) 아이콘이 표시됩니다. 편집 아이콘을 클릭하면 해당 블록의 설정 모달이 다시 열려 트리거 또는 작업 설정을 변경할 수 있습니다. 삭제 아이콘을 클릭하면 Automation에서 트리거 또는 작업 할당이 제거됩니다.

Automation 이름을 변경하려면 에디터 페이지 상단의 Automation 이름 옆에 있는 편집 아이콘을 클릭하세요.

## Automation 관리

목록의 각 Automation에는 **...** 버튼으로 접근하는 컨텍스트 메뉴가 있으며 네 가지 옵션을 제공합니다.

**Edit**는 If/Then 에디터에서 Automation을 열어 트리거, 작업 할당, 범위를 수정할 수 있습니다.

**Detail Info**는 Automation을 생성한 사람, 생성 시간, 마지막으로 수정한 사람, 마지막 수정 날짜 등 생성 및 수정 메타데이터를 표시합니다.

**History**는 Automation의 실행 기록을 열어 과거 실행 로그를 표시합니다. 문제를 진단하거나 예약된 Automation이 예상대로 실행되었는지 확인하는 데 유용합니다.

**Delete**는 Automation을 영구적으로 삭제합니다. 이 작업은 되돌릴 수 없습니다.

## 저장

Automation 에디터에서 변경한 후 우측 상단의 **Save** 버튼을 클릭하여 설정을 저장합니다. Automation은 저장되고 Automations 목록에서 토글이 활성화된 후에야 적용됩니다.
