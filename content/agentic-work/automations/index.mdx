# Automations

## 개요

Automation은 트리거와 작업을 연결하여 수동 개입 없이 작업이 실행되도록 합니다. 각 Automation은 **If / Then** 구조를 따릅니다. Automation을 활성화하는 조건을 정의하고, 해당 조건이 충족될 때 실행되어야 할 작업을 지정합니다. Automation은 Agentic Work 섹션의 Automations 탭에서 관리됩니다.

---

## Automations 목록

자동화 탭을 열면 두 섹션이 표시됩니다: **My Automations**와 **Team Automations**.

**My Automations**에는 사용자가 생성했거나 편집 권한이 있는 모든 자동화 작업이 표시됩니다. 목록의 각 항목은 자동화 이름, 진행 상태 표시기, 마지막 실행 날짜, 범위를 보여줍니다. 각 항목 왼쪽의 토글 스위치는 자동화의 활성화 여부를 제어합니다. 토글이 켜져 있으면 자동화가 활성 상태이며 트리거 조건이 충족될 때 실행됩니다. 토글이 꺼져 있으면 일시 중지되어 실행되지 않습니다. 일부 항목에는 상태 배지가 표시될 수 있습니다 — 예를 들어 "실패" 배지는 가장 최근 실행에서 오류가 발생했음을 나타냅니다.

**Team Automations**에는 그룹의 다른 멤버가 생성한 자동화 작업이 표시됩니다. 보기 및 복제 권한이 있지만 직접 편집할 수는 없습니다. 테이블에는 자동화 이름, 진행 상태, 마지막 실행 날짜, 생성자가 표시됩니다.

각 섹션 위에는 이름으로 자동화를 필터링하는 검색창이 있습니다.

### 진행과정

각 자동화는 진행과정 열에 자동화의 설정 상태를 나타내는 아이콘 집합을 표시합니다. 이 아이콘들은 트리거가 설정되었는지, 조건이 연결되었는지, 작업이 할당되었는지를 나타냅니다. 이 중 하나라도 없는 자동화는 완전히 설정되지 않은 것이며 실행되지 않습니다.

### 캘린더 사이드바

자동화 목록 오른쪽에 날짜 기반 자동화 내역를 제공하는 캘린더 사이드바가 있습니다. 날짜를 선택하면 세 가지 카테고리로 구성된 자동화를 볼 수 있습니다: 실행 중, 실행 대기, 실행 완료. 이 사이드바는 개별 항목을 열지 않고도 일별 자동화 활동을 빠르게 파악할 수 있게 해줍니다.

## Automation 생성

새 자동화를 생성하려면 자동화 목록 우측 상단의 **+ 새 자동화** 버튼을 클릭합니다. 빈 If/Then 캔버스가 있는 Automation 에디터가 열립니다. 페이지 상단의 기본 이름 옆에 있는 편집 아이콘을 클릭하여 자동화의 이름을 변경할 수 있습니다.

에디터는 세 가지 주요 영역으로 구성됩니다: 트리거를 정의하는 상단의 **If** 섹션, 실행할 작업을 할당하는 하단의 **Then** 섹션, 헤더의 **적용 범위** 드롭다운.

## 컨디션 정의 (If)

If 섹션의 **+ 자동화가 활성되는 조건 추가** 버튼을 클릭하여 트리거 유형을 선택합니다. 두 가지 트리거 유형이 있습니다: **Event**와 **Schedule**.

### 이벤트 트리거

이벤트 트리거는 Ontology 데이터 내에서 무언가가 발생했을 때 자동화를 실행합니다. 이벤트를 선택하면 두 가지 필드가 있는 설정 모달이 열립니다: **이벤트 종류**와 **오브젝트 선택**.

이벤트 종류는 자동화를 활성화할 변경의 종류를 결정합니다. 현재 사용 가능한 이벤트 유형은 "New row created"로, 선택된 Ontology 객체에 새 행이 삽입될 때마다 실행됩니다.

오브젝트 선택은 모니터링할 Ontology 객체를 선택합니다. 선택된 객체에서 지정된 이벤트가 발생하면 자동화가 트리거되어 할당된 작업을 실행합니다.

저장하면 If 블록에 이벤트 설정이 요약으로 표시됩니다 — 예를 들어 "New row created"와 객체 이름이 함께 표시됩니다.

### 스케줄 트리거

스케줄 트리거는 정의된 시간 간격에 따라 자동화를 실행합니다. 스케줄을 선택하면 실행 주기 설정 모달이 열립니다.

**실행 주기**는 자동화가 실행되는 빈도를 결정합니다. 세 가지 빈도 옵션이 있습니다: **Hourly**, **Daily**, **Weekly**.

**Hourly**로 설정하면 간격 값(예: 매 1시간 또는 매 3시간)을 지정합니다. 선택적인 **시간 지정** 토글을 사용하여 각 시간 내에서 자동화가 실행되어야 할 정확한 분을 정의할 수 있습니다.

**Daily**로 설정하면 간격 값(예: 매 1일 또는 매 2일)을 지정합니다. **시간 지정** 토글을 활성화하면 시간 필드와 타임존 선택기가 나타나 자동화가 실행되어야 할 정확한 시간(예: Asia/Seoul 기준 오전 08:30:00)을 설정할 수 있습니다.

**Weekly**로 설정하면 각 요일(일요일~토요일)에 대한 체크박스로 자동화가 실행이 될 요일을 선택합니다. 시간 필드와 타임존 선택기를 통해 선택된 각 날짜의 정확한 실행 시간을 정의할 수 있습니다.

저장하면 If 블록에 스케줄 요약이 표시됩니다 — 예를 들어 "Every 1 hour" 또는 "Every day at 08:30 AM".

## 작업 지정 (Then)

Then 섹션의 **+ 자동화가 실행할 워크 추가** 버튼을 클릭해 자동화가 실행할 **워크플로우**를 선택합니다. 선택 패널에는 접근 가능한 모든 워크플로우가 **AOS Workflows**, **My Workflows**, **Team Workflows** 세 가지 카테고리로 표시됩니다 — 워크플로우 목록 페이지와 동일한 구성입니다.

### Work 액션 설정

워크플로우를 선택하면 설정 모달이 열립니다.

워크는 배포된 워크플로우 목록이 표시되는 필수 드롭다운입니다. 각 항목에는 워크플로우 이름과 키값이 표시되어, 원하는 작업을 쉽게 찾을 수 있습니다. 자동화가 동작하려면 반드시 여기서 워크플로우를 선택해야 합니다.

**추가입력**은 선택 사항인 체크박스로, 활성화하면 지시문이나 컨텍스트를 입력할 수 있는 텍스트 영역이 나타납니다. 여기에 입력한 텍스트는 실행 시점에 워크플로우로 전달됩니다. 동일한 워크플로우가 컨디션에 따라 다른 시나리오를 처리해야 할 때 유용합니다.

**실행 결과 이메일로 받기**는 선택적 체크박스로, 활성화하면 해당 자동화의 생성자에게 실행 결과가 포함된 이메일 알림이 발송됩니다. 무인으로 실행되는 자동화 작업에 특히 유용하며, 플랫폼을 직접 확인하지 않고도 결과를 파악할 수 있습니다.

**저장**를 클릭해 설정을 저장합니다. 저장되면 Then 블록에 선택한 워크플로우 이름이 표시됩니다.

## 적용 범위

모든 자동화에는 에디터 우측 상단의 **적용 범위** 드롭다운으로 제어되는 범위 설정이 있습니다.

**나에게만**은 자동화를 제작자 본인만 볼 수 있도록 제한합니다. My Automations 목록에만 표시됩니다.

**그룹 전체**는 제작자가 소속된 그룹의 모든 멤버가 자동화를 볼 수 있게 합니다. 이 범위로 설정하면 다른 그룹 멤버의 Team Automations 섹션에 표시되어 보기 및 복제 권한이 부여됩니다.

## Automation 편집

If 블록과 Then 블록 모두 설정 요약 옆에 편집(연필) 및 삭제(휴지통) 아이콘이 표시됩니다. 편집 아이콘을 클릭하면 해당 블록의 설정 모달이 다시 열려 트리거 또는 작업 설정을 변경할 수 있습니다. 삭제 아이콘을 클릭하면 자동화에서 컨디션 또는 작업 할당이 제거됩니다.

자동화의 이름을 변경하려면 에디터 페이지 상단의 자동화 이름 옆에 있는 편집 아이콘을 클릭하세요.

## Automation 관리

목록의 각 자동화에는 **...** 버튼으로 접근하는 컨텍스트 메뉴가 있으며 네 가지 옵션을 제공합니다.

**편집**은 에디터에서 해당 자동화를 열어 트리거, 작업 할당, 범위를 수정할 수 있습니다.

**상세 정보**는 자동화를 생성한 사람, 생성 시간, 마지막으로 수정한 사람, 마지막 수정 날짜 등 생성 및 수정 메타데이터를 표시합니다.

**기록**은 자동화의 실행 기록을 열어 과거 실행 로그를 표시합니다. 문제를 진단하거나 예약된 자동화가 예상대로 실행되었는지 확인하는 데 유용합니다.

**삭제**는 자동화를 영구적으로 삭제합니다. 이 작업은 되돌릴 수 없습니다.

## 저장

Automation 에디터에서 변경한 후 우측 상단의 **저장** 버튼을 클릭하여 설정을 저장합니다. 자동화는 저장되고 Automations 목록에서 토글이 활성화된 후에야 적용됩니다.
